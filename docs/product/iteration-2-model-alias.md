## 1. 背景

AI Relay 当前的模型别名映射是**硬编码**在 `src/lib/providers/resolver.ts` 的静态对象：

```typescript
const MODEL_ALIASES: Record<string, string> = {
  'gpt-latest': 'gpt-5.4',
  'claude-sonnet': 'claude-sonnet-4-6',
  // ...
};
```

这带来三个问题：

1. **不可运维** — 修改别名需要改代码、重新部署
2. **不可扩展** — 用户无法添加自己的别名（如公司内部命名 `my-fast-model` → `gpt-5.4-mini`）
3. **模型管理缺失** — 没有批量导入、没有低频模型隐藏，Admin 面板看不到完整模型列表

圆桌讨论（rt_262195df）已确认迭代二的范围和方案。

## 5. 核心功能

### 5.1 模型别名映射表

#### 目标

将硬编码的 `MODEL_ALIASES` 迁移到 KV 存储，通过 Admin 面板可视化管理。

#### 数据模型

**KV Key**: `relay:models:aliases`

**Value**: JSON 对象 `{ [alias: string]: string }`

```json
{
  "gpt-latest": "gpt-5.4",
  "gpt-fast": "gpt-5.4-mini",
  "claude-sonnet": "claude-sonnet-4-6",
  "fast": "gpt-5.4-mini",
  "smart": "claude-sonnet-4-6"
}
```

#### 缓存策略

| 层 | TTL | 说明 |
|----|-----|------|
| 内存缓存 | 5 分钟 | Edge Function 内 `Map<string, {data, expiresAt}>` |
| KV 持久层 | 永久 | 写入即生效，读取走缓存 |

**解析优先级**：内存缓存 → KV 读取 → 硬编码兜底（保留原有默认别名作为 fallback）

#### Admin UI — 别名表格

- **入口**：侧边栏「模型配置」→「别名管理」标签
- **布局**：宽松行高 + 斑马纹表格
- **列**：别名 / 目标模型 / 来源（系统默认 / 用户自定义）/ 操作
- **操作列**：悬停显示「编辑」「删除」按钮
- **新增**：表格底部固定一行，点击「+ 添加别名」展开编辑行
- **编辑**：行内编辑，别名和目标模型均为可编辑输入框
- **删除**：二次确认弹窗（「确认删除别名 `fast` → `gpt-5.4-mini`？」）

#### 别名规则

| 规则 | 说明 |
|------|------|
| 唯一性 | 别名不可重复，冲突时输入框变红 + 提示「别名已存在」 |
| 格式 | 别名仅允许 `a-z 0-9 - _`，不区分大小写（存储时 lowercase） |
| 系统别名 | 标记为「系统默认」的别名不可删除，可编辑（用户覆盖后标记变为「用户自定义」） |
| 数量上限 | 单租户最大 200 条别名 |
| 目标模型 | 必须是当前已注册的模型 ID（下拉选择或自由输入 + 自动补全） |

#### 验收标准

- [ ] Admin 面板可查看所有别名（系统 + 用户自定义）
- [ ] 可新增别名，保存后立即生效
- [ ] 可编辑已有别名的目标模型
- [ ] 可删除用户自定义别名（系统别名不可删）
- [ ] 别名冲突时有明确错误提示
- [ ] 请求 `POST /v1/chat/completions` 使用别名时正确解析为目标模型
- [ ] KV 缓存 5 分钟，修改后最多 5 分钟生效（或主动刷新）
- [ ] 保留硬编码默认别名作为 KV 为空时的 fallback

### 5.2 CSV 批量导入/导出

#### 目标

支持通过 CSV 文件批量导入/导出模型配置（别名 + 隐藏标记），降低大量模型的配置成本。

#### CSV 格式规范

**文件名建议**: `ai-relay-models-YYYY-MM-DD.csv`

**列定义**:

| 列名 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `alias` | 是 | string | 别名（`a-z 0-9 - _`） |
| `target_model` | 是 | string | 目标模型 ID |
| `hidden` | 否 | boolean | 是否隐藏（`true`/`false`，默认 `false`） |
| `note` | 否 | string | 备注（仅 CSV 中保留，不存入 KV） |

**示例**:

```csv
alias,target_model,hidden,note
gpt-latest,gpt-5.4,false,系统默认
fast,gpt-5.4-mini,false,团队约定
smart,claude-sonnet-4-6,false,团队约定
legacy-model,gpt-5.4-nano,true,已弃用
```

#### 导入流程

1. **入口**：别名表格右上角「导入 CSV」按钮
2. **选择文件**：文件选择器，限制 `.csv` 文件，最大 50KB
3. **预览阶段**：
   - 解析 CSV 并展示预览表格（前 10 行）
   - 显示统计：新增 N 条 / 更新 N 条 / 跳过 N 条 / 错误 N 条
   - 错误行标红，hover 显示原因（格式错误、别名冲突、目标模型不存在）
4. **确认导入**：
   - 选择导入模式：「追加」（仅新增）/「覆盖」（全量替换）
   - 二次确认弹窗：「将新增 15 条、更新 3 条、跳过 2 条别名，确认？」
5. **导入结果**：Toast 提示成功/失败数量，表格自动刷新

#### 导出流程

1. **入口**：别名表格右上角「导出 CSV」按钮
2. **行为**：导出当前所有别名配置为 CSV 文件
3. **文件名**: `ai-relay-models-{timestamp}.csv`
4. **包含**：所有别名（系统 + 用户自定义）+ 隐藏标记

#### 校验规则

| 校验 | 处理 |
|------|------|
| 别名格式非法 | 跳过该行，标记错误 |
| 别名重复（CSV 内部） | 保留第一行，后续标为重复 |
| 别名与已有冲突（追加模式） | 跳过，标记「已存在」 |
| 目标模型不在注册列表 | 跳过，标记「模型不存在」 |
| CSV 超过 200 行 | 拒绝导入，提示「最多 200 条」 |
| 文件超过 50KB | 拒绝，提示文件过大 |
| 编码非 UTF-8 | 尝试自动检测，失败则提示 |

#### 验收标准

- [ ] 点击「导入 CSV」可选择 .csv 文件
- [ ] 导入前展示预览和统计
- [ ] 追加模式：仅新增不覆盖已有
- [ ] 覆盖模式：全量替换（保留系统别名）
- [ ] 错误行有明确提示
- [ ] 导入成功后表格自动刷新
- [ ] 点击「导出 CSV」下载当前配置
- [ ] 导出文件可再次导入（round-trip 一致）

### 5.3 低频模型隐藏

#### 目标

允许管理员隐藏低频/弃用模型，减少模型列表噪音，让用户只看到常用模型。

#### 存储

与别名共用 KV Key `relay:models:aliases`，增加 `hidden` 标记：

```json
{
  "aliases": {
    "gpt-latest": "gpt-5.4",
    "fast": "gpt-5.4-mini"
  },
  "hidden": ["gpt-5.4-nano", "gpt-5.4-mini"]
}
```

> **向后兼容**：如果 KV 值是纯对象（旧格式），自动迁移为新结构。

#### Admin UI — 隐藏标记

- 别名表格增加「可见性」列（眼睛图标）
- 点击图标切换隐藏/显示状态
- 隐藏的模型行变为半透明 + 删除线
- 表格顶部增加筛选器：「全部 / 仅可见 / 仅隐藏」

#### 隐藏行为

| 场景 | 隐藏模型的表现 |
|------|---------------|
| `/v1/models` 列表 | 不出现在返回列表中 |
| `/v1/chat/completions` 请求 | **仍然正常处理**（隐藏 ≠ 禁用） |
| Admin 面板 | 显示为半透明行，可取消隐藏 |
| 别名指向隐藏模型 | 别名仍生效，但目标模型不在 `/v1/models` 中展示 |

> **关键设计**：隐藏只影响模型发现（`/v1/models`），不影响实际调用。已有代码中使用隐藏模型名的客户端不会中断。

#### 验收标准

- [ ] Admin 面板可标记模型为隐藏/可见
- [ ] 隐藏模型不出现在 `/v1/models` 响应中
- [ ] 直接调用隐藏模型名仍正常工作
- [ ] 别名指向隐藏模型时，别名解析正常
- [ ] 隐藏状态可通过 CSV 导入/导出

## 6. API 设计

### 6.1 Admin API — 别名管理

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/admin/aliases` | 获取所有别名（含隐藏标记） |
| `PUT` | `/api/admin/aliases` | 批量更新别名（覆盖写入） |
| `POST` | `/api/admin/aliases` | 新增单条别名 |
| `DELETE` | `/api/admin/aliases?alias=xxx` | 删除单条别名 |
| `POST` | `/api/admin/aliases/import` | CSV 导入（multipart/form-data） |
| `GET` | `/api/admin/aliases/export` | CSV 导出（返回 .csv 文件） |

#### GET /api/admin/aliases 响应

```json
{
  "aliases": {
    "gpt-latest": { "target": "gpt-5.4", "source": "system" },
    "fast": { "target": "gpt-5.4-mini", "source": "user" }
  },
  "hidden": ["gpt-5.4-nano"],
  "total": 15
}
```

#### POST /api/admin/aliases 请求

```json
{
  "alias": "fast",
  "target": "gpt-5.4-mini",
  "hidden": false
}
```

### 6.2 请求链路变更

`resolveModelAlias()` 函数改造：

```
1. 检查内存缓存（TTL 5min）→ 命中则返回
2. 读取 KV `relay:models:aliases` → 解析并缓存
3. KV 为空时 fallback 到硬编码 MODEL_ALIASES
4. 返回解析结果
```

**KV 预算**：+0 次/请求（内存缓存命中时）

### 6.3 /v1/models 变更

返回模型列表时过滤掉 `hidden` 列表中的模型 ID。

## 7. 技术约束

| 约束 | 应对 |
|------|------|
| Edge Runtime 内存缓存 | `Map` + TTL，5 分钟过期自动刷新 |
| Vercel KV 免费档 3000 次/日 | 内存缓存命中时不走 KV，稳态 +0 |
| KV 单 key 1MB 限制 | 200 条别名 ≈ 20KB，远低于限制 |
| Serverless 冷启动 | 首次请求读 KV（~50ms），后续缓存命中 |
| CSV 文件大小 | 前端限制 50KB / 200 行 |
| 向后兼容 | 保留硬编码 `MODEL_ALIASES` 作为 fallback |

## 8. 指标

| 指标 | 定义 | 目标 |
|------|------|------|
| 别名配置率 | 使用自定义别名的部署实例比例 | > 30% |
| CSV 导入成功率 | 导入操作中成功的比例 | > 95% |
| 别名解析延迟 | 别名解析增加的请求延迟（缓存命中） | < 1ms |
| KV 调用增量 | 每请求 KV 调用增加量 | +0（稳态） |

## 9. 风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 别名循环引用 | A→B→A 导致无限解析 | 解析时检测循环，最大深度 5 |
| 别名覆盖核心模型名 | 用户将 `gpt-5.4` 别名指向错误模型 | 系统别名不可删除，目标模型校验 |
| CSV 导入覆盖误操作 | 覆盖模式丢失已有配置 | 二次确认 + 导出备份提示 |
| 缓存不一致 | 修改后最多 5 分钟延迟 | 提供「立即刷新」按钮 |
| KV 值格式迁移 | 旧格式（纯对象）→ 新结构 | 读取时自动检测并迁移 |
